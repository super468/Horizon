## [Coelanox: Auditable ML Inference Runtime in Rust](https://www.coelanox.com/) ⭐️ 8.0/10

Coelanox is a new Rust-based inference runtime that packages ML models in sealed .cnox containers with SHA-256 cryptographic verification and per-operation audit logging, enabling full traceability of what operations executed during inference. This addresses a critical gap in production ML systems where frameworks like PyTorch and ONNX Runtime cannot verify what operations actually ran or provide cryptographic proof of inference results—essential for compliance, debugging, and reproducibility in regulated industries. The runtime currently supports a scalar backend as the reference implementation, with plans for SIMD and GPU acceleration. Every inference run generates a cryptographically linked audit log containing operation type, output tensor SHA-256 hash, and output samples—a BERT model example demonstrates the system working with synthetic inputs.

rss · Hacker News - Show HN · Apr 18, 20:37

**Background**: ML inference auditability has become a compliance requirement in finance, healthcare, and other regulated industries. Unlike traditional frameworks that only provide final outputs, Coelanox creates a verifiable chain linking inputs, operations, and outputs through cryptographic hashes. The .cnox container format ensures model integrity via SHA-256 verification before any computation begins, while the audit log provides per-operation visibility that can be used for debugging, compliance audits, and reproducibility.

## [PgQue: Kafka-Style Postgres Message Queue](https://github.com/NikolayS/pgque) ⭐️ 7.0/10

PgQue is a Kafka-like message queue implementation built on Postgres that uses event stream patterns rather than traditional job queue semantics, with consumer latency of 0.25ms and end-to-end latency between 1-2 seconds. This matters for developers who need lightweight queue solutions without deploying external infrastructure like Kafka or RabbitMQ, offering Postgres durability for outbox patterns and small fan-out use cases. PgQue brings back PgQ architecture, originally designed at Skype for hundreds of millions of users, and works on any Postgres platform including managed providers. Unlike SKIP LOCKED-based queues, it uses one event stream with multiple independent worker cursors.

hackernews · gmcabrita · Apr 18, 16:50

**Background**: Traditional job queues (like SQS, RabbitMQ) use worker-based pull semantics where workers take jobs from a shared pool. Event streams (like Kafka) use a different pattern where each consumer maintains its own cursor through an ordered log. PgQ was developed at Skype to handle massive-scale messaging and is one of the longest-running Postgres queue architectures in production.

## [Sostactic: SOS Proof Tactics for Lean 4 Polynomial Inequalities](https://github.com/mmaaz-git/sostactic) ⭐️ 7.0/10

Sostactic is a new Lean 4 package providing tactics for proving polynomial inequalities via sum-of-squares (SOS) decompositions, powered by a Python backend, significantly outperforming existing tactics like nlinarith and positivity. This addresses a real gap in Lean's nonlinear inequality proving capabilities, enabling proofs of global polynomial nonnegativity, nonnegativity over semialgebraic sets, and emptiness of semialgebraic sets that existing tactics cannot handle. The underlying theory connects 20th century real algebraic geometry (Hilbert's theorems on PSD=SOS) with 21st century semidefinite programming, using solvers like SDPA, MOSEK, or CVXPY in the Python backend to find SOS decompositions.

rss · Hacker News - Show HN · Apr 18, 22:36

**Background**: Sum-of-squares (SOS) decomposition is a method where a polynomial is expressed as a sum of squares of other polynomials, which guarantees the polynomial is nonnegative everywhere. Semialgebraic sets are sets defined by polynomial equalities and inequalities. The connection between SOS decompositions and semidefinite programming, established in the 2000s, made these techniques computationally practical.
