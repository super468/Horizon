## [Claude Writes Full FreeBSD Kernel RCE Exploit](https://github.com/califio/publications/blob/main/MADBugs/CVE-2026-4747/write-up.md) ⭐️ 8.0/10

Anthropic's Claude AI was given a CVE writeup for a FreeBSD kernel vulnerability (CVE-2026-4747) and successfully wrote a complete remote code execution exploit that achieves a root shell. This represents a groundbreaking demonstration of AI's capability in security research: not only can AI discover vulnerabilities, but it can also autonomously construct working exploits. This marks a significant milestone in AI-driven vulnerability research and has profound implications for both offensive and defensive cybersecurity. The exploit leverages the fact that FreeBSD 14.x lacks KASLR (Kernel Address Space Layout Randomization) and stack canaries for integer arrays (the overflowed buffer is int32_t[]), making kernel exploitation significantly easier than on modern Linux kernels. Notably, the same bug was originally discovered by Claude (Nicholas Carlini at Anthropic), and a subsequent instance was tasked with writing the exploit.

hackernews · ishqdehlvi · Apr 1, 05:21

**Background**: Remote Code Execution (RCE) is a class of vulnerabilities that allows attackers to execute arbitrary code on a target system remotely, often with elevated privileges. The Common Vulnerabilities and Exposures (CVE) system provides standardized identifiers for publicly known security vulnerabilities. Kernel-level vulnerabilities are particularly critical because they operate with the highest privilege level, and modern operating systems implement protections like KASLR and stack canaries to make exploitation more difficult. FreeBSD is a Unix-like operating system known for its advanced networking and security features.

NPM supply chain attacks typically follow three patterns: account hijacking (compromising maintainer credentials), typosquatting (creating similarly-named packages), and dependency confusion. This incident represents the account hijacking pattern targeting a critical library used by millions of developers.

This approach addresses a critical gap in AI evaluation by enabling direct comparison between task demands and model capabilities, and predicting performance on new tasks—something current benchmarks cannot do. It could transform how we assess, understand, and select AI models for specific applications. ADeLe scores both tasks and models across 18 cognitive scales, allowing researchers to decompose performance into interpretable components and understand what drives model success or failure. This enables reliable prediction of model performance on unseen tasks based on capability alignment.

They cannot explain failures or reliably predict outcomes on new tasks. ADeLe represents a shift from task-based to capability-based evaluation, enabling a more fundamental understanding of AI model strengths and limitations.
